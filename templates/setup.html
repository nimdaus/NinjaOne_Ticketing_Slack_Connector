NinjaOne API application requirements

Before completing setup, verify your NinjaOne API application (Administration → Apps → API) is configured as follows:

  • Grant type: Authorization Code — not Client Credentials or any other type
  • Scopes: monitoring, management, and offline_access
  • Redirect URI: the callback URL shown below must be registered exactly

Wrong grant type: token exchange returns HTTP 400 and setup fails.
Missing scopes: setup completes, but ticket API calls return 403 Forbidden. No refresh_token in response: usually means the grant type is wrong — only Authorization Code issues a refresh token.
